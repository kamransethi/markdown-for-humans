# Task: Fix Table Dragging Bug

## 1. Task Metadata
- **Task name:** Fix Table Dragging Creating Erroneous Rows/Cols
- **Slug:** table-drag-fix
- **Status:** shipped
- **Created:** 2026-03-06

## 2. Context & Problem
When users click and drag elements inside or around a table, the editor frequently creates accidental rows or columns, breaking the user's document structure.

## 3. Desired Outcome & Scope
- Drag-and-drop operations should strictly move content without mutating the structural dimensions of the table.
- Disable drag-based row/col insertion unless specifically dragging row/column headers (if Prosemirror table allows).
- Ensure the table manipulation commands (add row, add col) are strictly bound to explicit user actions, not unintended drop events.

## 4. Work Breakdown
- Investigate `tiptap` / `prosemirror-tables` drop handlers.
- Intercept drop events on table cells and block row/column creation if the drop source is external to table structure nodes.
- Validate drag handles for blocks and ensure they behave predictably around tables.
