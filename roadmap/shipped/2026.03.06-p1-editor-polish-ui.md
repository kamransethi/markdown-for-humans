## 2. Context & Problem
We are transitioning the editor from a POC to a fully-featured, high-quality product. Several minor UX issues disrupt the flow:
1. Toggling alerts is cumbersome; if an alert text is selected, pressing "Alert" doesn't disable it.
2. The H1, H2, H3 buttons clutter the toolbar.
3. The "All changes saved" bubble is too persistent and distracting.
4. Users have no control over how lists are formatted inside tables (`<br />-` vs `<ul><li>`), limiting markdown flavor compatibility.

## 3. Desired Outcome & Scope
1. **Alert Toggle:** Clicking "Alert" on selected text transforms it, but clicking it while the caret is inside an existing alert (or selecting existing alert text) removes the alert formatting and restores plain text.
2. **Heading Menu:** Combine H1, H2, H3 into a single `aA` dropdown menu button.
3. **Save Button:** Replace the distracting bubble with a dedicated Save icon in the toolbar. It should be disabled when there are no changes, and enabled when changes are pending (allowing the user to click to save).
4. **Table List Configuration:** Add a settings toggle to switch the editor between `<br />-` and `<ul>/<li>` list structures for table cells.

## 4. Work Breakdown
- Implement alert active-state detection and unwrapping logic in `BubbleMenuView` or the GitHub alerts extension.
- Create a drop-down toolbar component for the headings (`aA`) and map commands to H1/H2/H3.
- Adjust the save service to manage the disabled/enabled state of a new toolbar Save button, and remove the floating bubble notification.
- Add an extension setting for table list mode and handle the serialization appropriately during export.
