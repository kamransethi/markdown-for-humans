# Task: Robust Mermaid Chart Editing Experience

## 1. Task Metadata
- **Task name:** Mermaid Advanced Editor / Split-View
- **Slug:** mermaid-robust-editing
- **Status:** shipped
- **Created:** 2026-03-06

## 2. Context & Problem
- Current copying and pasting of Mermaid code into the editor's modal often accidentally pastes the code into the background editor itself.
- Visualizing complex syntax without instant feedback forces repeated switching.
- Concept is outlined in older `task-p0-mermaid-split-view.md`.

## 3. Desired Outcome & Scope
- **Robust Input Handling:** The modal or split-editor must perfectly isolate copy/paste and prevent events from falling through to the main Prosemirror instance.
- **Split View Alternative:** Rather than a popup modal, the user should be able to click on a diagram and open a split view (code on left, live output on right).
- Fix bug where pasting pastes in the background.

## 4. Work Breakdown
- Option A: Fix the React/DOM event bubbling in the current modal so `paste` events run `e.stopPropagation()`.
- Option B (Preferred): Implement the split-view architecture described in prior specs, rendering the raw text block in the Tiptap node view beside a live-refreshing Mermaid container.
