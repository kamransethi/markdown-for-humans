# Task: Content Insertion via Quick Trigger Patterns

## 1. Task Metadata
- **Task name:** Input Rules (Markdown Trigger Patterns)
- **Slug:** input-rules-regex
- **Status:** shipped
- **Created:** 2026-03-06

## 2. Context & Problem
Users expect modern rich-text editors to convert markdown formatting on the fly based on specific typing patterns. While some native Tiptap rules exist, they are incomplete based on user requirements.

## 3. Desired Outcome & Scope
Implement strict InputRules to transform current line into specific blocks based on the following regexes:
- **Heading 1:** `/^#\s$/` -> Converts to H1
- **Heading 2:** `/^##\s$/` -> Converts to H2
- **Bullet List:** `/^\*\s$/` or `/^-\s$/` -> Unordered list
- **Checkbox/Task:** `/^\[\]\s$/` -> Task list item
- **Code Block:** `/^` ` ` `\s$/` -> Code block node
- **Blockquote:** `/^>\s$/` -> Blockquote node

## 4. Work Breakdown
- Check existing Tiptap `InputRule` configurations.
- Map the defined Regex triggers to the respective node creation commands.
- Ensure triggers only activate at the start of a block/line.
