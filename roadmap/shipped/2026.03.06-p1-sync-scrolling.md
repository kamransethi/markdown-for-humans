# Task: Editor Side-by-Side Sync Scrolling

## 1. Task Metadata
- **Task name:** Sync Scrolling for Side-by-Side View
- **Slug:** sync-scrolling
- **Status:** shipped
- **Created:** 2026-03-06

## 2. Context & Problem
In the side-by-side view, the Prosemirror-based rich text editor and the raw markdown source editor scroll independently. Users modifying raw markdown cannot automatically see their concurrent location in the visual editor, interrupting workflows.

## 3. Desired Outcome & Scope
- When scrolling the raw markdown view, the visual editor should scroll to the corresponding position.
- When scrolling the visual editor, the raw markdown view should scroll to match.
- The scrolling should feel 1:1, achieved by mapping the document line numbers or node positions between the two editors.

## 4. Work Breakdown
- Establish a scroll event listener on both `tiptap` and the source editor.
- Convert `tiptap` node positions to line numbers.
- Calculate the percentage or relative vertical offset to synchronize scroll heights safely.
- Add configuration to enable/disable synchronized scrolling manually if desired.
